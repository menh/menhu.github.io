C++ 指针传参自我剖析

结论：指针传递和引用传递都可以用来修改参数的值，不同之处在于，引用传递传递的是自己，只有一个副本，改变的是自己，对形参所有的操作都是在原实参所在的地址进行。指针传递传递会进行自我拷贝，两个副本指向同一地址，改变的是指向地址的内容。对形参所做的操作
可能是实参或形参(不进行重新分配内存的情况下)所指向的地址，也可能是形参所在或新建的地址。

引用和指针在存放容器的区别

问题来源
在看框架代码时，遇到一段类似这样的代码


struct ST_DATA{
    string key;
    int num;
};
class CDataCache{
    public:
        void append(ST_DATA &p_stData){
            vecDataCache.push_back(p_stData);
        }
    private:
        vector<ST_DATA>vecDataCache;
};

看到这段，主要是对append函数的疑惑，引用传递结构体ST_DATA，然后将数据放到vector容器中。
